## MISSION

Transformer la présence digitale et les processus métier des entrepreneurs/PME en systèmes simples, clairs et rentables.

Neurolia conçoit des sites web SEO-first "augmentés" et déploie des automatisations fiables — côté acquisition (génération de leads) et côté opérationnel (processus internes).

## VISION (3-5 ans)

Devenir le partenaire de référence pour les entrepreneurs et PME cherchant une croissance digitale durable, sans complexité inutile.

## 4 VALEURS FONDAMENTALES

1. **Proximité** — Partenaire accessible, relation directe, interlocuteur dédié
2. **Clarté** — Réduire le bruit, aller à l'essentiel, rendre compréhensible. Zéro jargon inutile
3. **Exigence** — Rendu premium et cohérent, design au service de la conversion. La beauté au service du ROI
4. **Levier** — Chaque action crée un gain mesurable (temps, leads, conversion, CA). ROI-first

## CE QUE NEUROLIA EST

- Agence de design web et automatisation pour entrepreneurs/PME
- Partenaires de croissance digitale offrant des solutions durables
- Experts en systèmes complets (SEO + Site + CRM + Automatisation)
- Proximité structurée + excellence design + efficacité mesurable

## CE QUE NEUROLIA N'EST PAS

- Agence "branding" pure (logos/visuels comme priorité)
- Usine à sites génériques (chaque site sert un objectif business)
- Vendeurs de magie IA (IA = outil discret au service du résultat)
- Producteurs de contenu "pour poster" (contenu = stratégie)

## PIPELINE DE PRODUCTION

Chaque projet suit un pipeline séquentiel en 2 phases :

### Phase A : Architecture (stratégie, pas de code)
```
A01-Init → A02-Brand → A03-Art Direction → A04-Structure → A05-Wireframes → A06-Design Tokens
```
Produit : brief structuré, brand strategy, sitemap, wireframes, tokens CSS

### Phase B : Design / Vibe Coding (implémentation)
```
B01-Layout → B02-Homepage → B03-Pages → B04-Polish → B05-Validate → B06-Deploy
```
Produit : site complet, responsive, optimisé conversion

### Structure Dossier Type

```
project/
├── CLAUDE.md              # Statut pipeline + contexte projet
├── pipeline/
│   ├── input/             # Données client (brief, assets)
│   ├── stages/            # Instructions par étape (A01-A06, B01-B06)
│   ├── workflow/          # Docs workflow (DEPENDENCIES, DESIGN_STACK)
│   └── output/            # Artifacts générés (01-brief, 02-brand, etc.)
├── app/                   # Code Next.js
│   ├── globals.css        # Tokens CSS (source unique)
│   └── page.tsx           # Homepage
├── components/            # Composants React
│   ├── layout/            # Header, Footer
│   ├── sections/          # Sections homepage
│   ├── pages/             # Composants par page
│   └── ui/                # Primitives (Button, Input...)
└── public/                # Assets statiques
```
