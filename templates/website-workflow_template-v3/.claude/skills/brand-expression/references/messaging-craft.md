# Messaging Craft — Reference

## Mapping Plateforme → Messages par Section

Chaque section du site porte un message specifique derive d'un composant de la plateforme.

### Matrice de derivation

| Section type | Composant plateforme source | Type de message |
|-------------|---------------------------|-----------------|
| Hero | Insight + Promise | Tension → resolution (tagline + baseline) |
| A propos | Purpose + Vision + Values | Identite narrative |
| Services/Offre | Mission + Proof Points | Benefice concret par categorie |
| Temoignages | Proof Points | Social proof contextualise |
| Contact | Promise + Insight | Invitation a agir (CTA final) |
| FAQ | Proof Points + Values | Reponses qui renforcent le positionnement |
| Footer | Brand Essence | Rappel identitaire (1 phrase) |

### Process par section

Pour chaque section identifiee dans 00-brief.md > Pages :

1. **Identifier** le composant plateforme source (voir matrice)
2. **Extraire** les mots-cles de la Language Library pertinents
3. **Appliquer le process creatif** (5 etapes) au message de cette section
4. **Verifier** la coherence avec les messages des sections adjacentes

## CTAs — Methodologie

### Hierarchie CTA

| Niveau | Fonction | Exemples patterns |
|--------|----------|-------------------|
| **Primaire** | Action principale du site (KPI) | "Appelez-nous", "Reservez", "Demander un devis" |
| **Secondaire** | Exploration / engagement | "Voir la carte", "Decouvrir nos services" |
| **Tertiaire** | Micro-actions contextuelles | "En savoir plus", "Voir les details" |

### Regles CTAs

1. **Verbe d'action en premier** : "Reservez votre table" pas "Votre table vous attend"
2. **Specifique au contexte** : "Voir le menu du jour" pas "En savoir plus"
3. **Coherent avec l'archetype** :
   - Hero/Ruler → verbes forts ("Agissez", "Prenez le controle")
   - Caregiver → verbes doux ("Contactez-nous", "Parlons-en")
   - Jester → verbes ludiques ("Testez", "Lancez-vous")
4. **Pas de generique** : "Cliquez ici", "Soumettre", "Envoyer" sont BANNIS
5. **Longueur** : 2-4 mots ideaux, max 6

### Process creatif CTA

Appliquer les 5 etapes mais avec variantes reduites (5 au lieu de 8) et scoring simplifie (Specificite + Coherence ADN seulement — les CTAs sont utilitaires, pas poetiques).

## USPs — Construction

Les USPs (Unique Selling Propositions) dans positioning.md derivent de :
- **Proof Points** (00-platform.md) → faits concrets
- **Discriminator** (00-platform.md) → ce qui est unique
- **Benefits** (00-platform.md) → benefices fonctionnels + emotionnels

### Format USP

```
**[Benefice en 3-5 mots]** : [Preuve concrete en 1 phrase]
```

### Regles

1. **3 USPs exactement** — pas 2, pas 5
2. **Chaque USP couvre un angle different** : ex. expertise + proximite + resultat
3. **Preuves verifiables** — pas "des annees d'experience" mais "12 ans dans le quartier"
4. **Ordre** : du plus differentiant au plus attendu

## Taglines de Categories (services.md)

Chaque categorie de service a sa propre tagline courte.

### Process adapte

1. **Lire** la categorie dans 00-brief.md
2. **Identifier** la valeur ajoutee specifique de cette categorie
3. **Appliquer** le process creatif avec 5 variantes (pas 8 — scope plus petit)
4. **Critere supplementaire** : la tagline de categorie ne doit PAS etre confondue avec la tagline principale

### Coherence inter-categories

Verifier que les taglines de categories forment un ensemble coherent :
- [ ] Meme registre (pas une formelle et une familiere)
- [ ] Meme longueur approximative (pas 3 mots et 12 mots)
- [ ] Pas de repetition de mots entre categories
- [ ] Ensemble lisible comme une "collection"

## Headlines par Section — Principes

Les messages par section dans positioning.md suivent un arc narratif :

```
Hero (accroche/tension)
  → Services (resolution/offre)
    → A propos (identite/confiance)
      → Contact (invitation/action)
```

### Verification de l'arc

- [ ] Le Hero pose une tension OU une promesse forte
- [ ] Les Services repondent a la tension avec du concret
- [ ] Le A propos humanise et rassure
- [ ] Le Contact relance l'urgence ou la desirabilite
- [ ] Aucune section ne repete le message d'une autre
